C10K problem

C10K问题在1999年被提出，随着技术的发展，又出现了C100K，C10M等等相近的问题，不过问题本质相同。

这个问题的本质在于如何优化网络套接字，从而让服务器可同时处理大量的客户端请求。

并发连接和每秒请求数(QPS)不同。QPS可以通过提高处理速度（吞吐量）来提高。而并发连接数则要靠对于资源调度来实现。
